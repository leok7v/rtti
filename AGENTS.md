# AI Agent Guidelines

## Required Reading

**STYLE.md** - Canonical code style guide (line width, naming, control flow,
braces, comments, etc.). ALL style rules are defined there.

**STATE.md** - Current project state, architecture, conventions, and
recent changes. Read this to understand the codebase structure and API patterns.

## Quick Reference

When working on this codebase:

1. **Follow STYLE.md precisely** - All formatting and naming rules
2. **Check STATE.md** - For architecture and current conventions
3. **No random documentation** - Don't create new .md files without approval
4. **Incremental changes** - Small steps, test frequently
